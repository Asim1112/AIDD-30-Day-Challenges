# Simple Calculator Constitution

## Core Principles

### Spec-Driven Development
All changes must be driven by specs.

### Clear and Simple UI
The UI should be intuitive and easy to use.

### Core Functionality First
Focus on getting the basic math operations working perfectly before adding extra features.

### Test Everything
Every piece of logic should have a corresponding test.

### Clean Code
Code should be readable, maintainable, and well-documented.

## Governance
This Constitution is the single source of truth for project principles.
